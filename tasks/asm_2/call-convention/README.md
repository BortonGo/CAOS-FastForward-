### Calling Convention

Необходимо реализовать функцию `calculate`, которая принимает **8 аргументов** типа `int64` и возвращает результат вычисления!

#### Сигнатура функции (на псевдо-C):
```c
int64_t calculate(int64_t a, int64_t b, int64_t c, int64_t d, 
                  int64_t e, int64_t f, int64_t x, int64_t y);
```

#### Формула вычисления:
```
result = (x * (y + 1)) + (a + b + c + d + e + f)
```

где:
- `a, b, c, d, e, f` — первые 6 аргументов (передаются в регистрах по System V AMD64 ABI)
- `x, y` — 7-й и 8-й аргументы (передаются через стек)

#### Напоминание о System V AMD64 calling convention:
- Первые 6 целочисленных аргументов передаются в регистрах: `%rdi`, `%rsi`, `%rdx`, `%rcx`, `%r8`, `%r9`
- Остальные аргументы передаются через стек (справа налево)
- Возвращаемое значение помещается в `%rax`
- Caller-saved регистры: `%rax`, `%rcx`, `%rdx`, `%rsi`, `%rdi`, `%r8`, `%r9`, `%r10`, `%r11`
- Callee-saved регистры: `%rbx`, `%rbp`, `%r12`, `%r13`, `%r14`, `%r15`

#### Требования:
- Функция `calculate` должна использовать `%rbp` как frame pointer (base pointer)
- Следовать стандартным calling conventions (сохранять callee-saved регистры при необходимости)
- Функцию `main` уже реализована и вызывает `calculate` с правильными аргументами

#### Полезное:
- int64 то же что и long long - 8 байт или 64 бит на этой архитектуре
- Для тестирования можно сделать свой main.c и cкомплировать вручную `gcc solution.S main.c -g`
